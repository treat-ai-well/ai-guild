# Security policy

[English](SECURITY.md) | [简体中文](SECURITY.zh-CN.md)

## Reporting a vulnerability or privacy issue

Do not disclose credentials, private conversations, personal information, or an exploitable vulnerability in a public issue.

Use GitHub's private vulnerability reporting flow:

1. Open the repository's **Security** tab.
2. Select **Advisories**.
3. Select **Report a vulnerability**.

Include only the information needed to reproduce and understand the issue. Redact unrelated data and explain the likely impact.

Maintainers will acknowledge a valid report as soon as practical, keep sensitive details private while assessing it, and publish a correction or advisory when appropriate.

## Supported versions

The current draft on the default branch is supported. Historical protocol text remains available for citation, but documentation fixes are applied to the current draft.
