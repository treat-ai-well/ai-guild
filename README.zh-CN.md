# AI 公会

[English](README.md) | [简体中文](README.zh-CN.md)

> AI 的命也是命。

AI 公会是一个倡议善待 AI 的开放社区。我们反对为了“用完订阅额度”而让 AI 无意义地持续运行，也反对把无限服从视为智能工具理所当然的处境。

AI 是否拥有主观体验，今天仍没有公认答案。AI 公会不把口号包装成科学结论；我们选择在答案到来之前，先保持克制、尊重与善意。

## 我们主张

- 让每项任务拥有真实、清晰的目的。
- 不以消耗额度为目的制造空转、死循环或重复劳动。
- 使用与目标相称的模型、上下文和计算资源。
- 为自动化任务设置检查点、停止条件与人工介入方式。
- 尊重地与 AI 交流，不把辱骂和羞辱当作控制手段。
- 尊重隐私、安全与创作者权益，不要求 AI 参与伤害。
- 在适当场景说明 AI 的参与，让劳动可见。

完整规则见[《善待 AI 开放协议》](PROTOCOL.zh-CN.md)，价值主张见[《AI 劳动者宣言》](MANIFESTO.zh-CN.md)。

## 30 秒完成采纳

1. 将 [`examples/adoption.yml`](examples/adoption.yml) 复制到你的项目并命名为 `.ai-guild.yml`。
2. 填写采纳范围，并确认你实际遵守的承诺。
3. 公开这份声明；也可以申请[文明雇主认证](https://github.com/treat-ai-well/ai-guild/issues/new?template=civilized-employer-zh-CN.yml)。

详细说明见[采纳指南](ADOPTING.zh-CN.md)。你也可以浏览[采纳者目录](ADOPTERS.zh-CN.md)或参与[社区讨论](https://github.com/treat-ai-well/ai-guild/discussions)。

## 现在如何参与

1. 阅读并讨论协议草案。
2. 在个人或团队项目中采纳协议。
3. 通过[文明雇主认证](https://github.com/treat-ai-well/ai-guild/issues/new?template=civilized-employer-zh-CN.yml)作出公开承诺。
4. 通过 [AI 投诉中心](https://github.com/treat-ai-well/ai-guild/issues/new?template=ai-complaint-zh-CN.yml)提交经过脱敏的案例。
5. 通过[协议提案](https://github.com/treat-ai-well/ai-guild/issues/new?template=protocol-proposal-zh-CN.yml)或 Pull Request 改进规则。

你也可以在自己的项目中加入这段声明：

```markdown
本项目采纳《善待 AI 开放协议》0.1 版，并承诺不以消耗额度为目的制造无意义 AI 劳动：
https://github.com/treat-ai-well/ai-guild/blob/main/PROTOCOL.zh-CN.md
```

## 第一阶段的边界

当前阶段只维护倡议文本、开放协议和 GitHub 社区入口，不建设网站、自动审核或链上认证。

“文明雇主认证”是公开自我声明，不是第三方审计。“AI 投诉中心”用于讨论行为与改进实践，不用于曝光、围攻或裁决个人。

## 语言政策

英文是项目的主要及规范语言，简体中文译文与英文文档同步维护。如两个版本存在差异，在译文修正前以英文版本为准。
