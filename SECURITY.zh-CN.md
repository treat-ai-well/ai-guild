# 安全政策

[English](SECURITY.md) | [简体中文](SECURITY.zh-CN.md)

## 报告漏洞或隐私问题

不要在公开 Issue 中披露凭据、私密对话、个人信息或可被利用的漏洞。

请使用 GitHub 的私密漏洞报告流程：

1. 打开仓库的 **Security** 标签页。
2. 选择 **Advisories**。
3. 选择 **Report a vulnerability**。

只提供复现和理解问题所需的信息，删除无关数据，并说明可能造成的影响。

维护者会在合理时间内确认有效报告，在评估期间保护敏感细节，并在适当时发布修正或公告。

## 支持版本

默认分支上的当前草案受支持。历史协议文本会保留以便引用，但文档修正只应用于当前草案。
