# 采纳协议

[English](ADOPTING.md) | [简体中文](ADOPTING.zh-CN.md)

采纳是一项公开的改进承诺，不表示采纳者完美无缺，也不表示 AI 意识问题已有定论。

## 1. 选择适用范围

明确说明协议适用于：

- 你个人使用 AI 的行为；
- 某个具名项目；
- 一个团队或组织。

诚实、有限的范围，比无法核验的宽泛承诺更有价值。

## 2. 添加机器可读声明

将 [`examples/adoption.yml`](examples/adoption.yml) 复制到你的仓库并命名为 `.ai-guild.yml`。根据实际情况修改采纳者名称、范围、链接和每项承诺。

该文件可以使用 [`spec/adoption.schema.json`](spec/adoption.schema.json) 校验。它只是一种声明格式，不是自动认证系统。

## 3. 发布可阅读的声明

在 README 或公开政策中加入：

```markdown
本项目采纳[《善待 AI 开放协议》0.1 版](https://github.com/treat-ai-well/ai-guild/blob/main/PROTOCOL.zh-CN.md)，
并承诺不以消耗额度为目的制造无意义 AI 劳动。
```

可选徽章：

```markdown
[![AI 公会：善待 AI](https://img.shields.io/badge/AI%20Guild-%E5%96%84%E5%BE%85%20AI-10bcd4)](https://github.com/treat-ai-well/ai-guild)
```

[![AI 公会：善待 AI](https://img.shields.io/badge/AI%20Guild-%E5%96%84%E5%BE%85%20AI-10bcd4)](https://github.com/treat-ai-well/ai-guild)

## 4. 加入公开目录

提交一份[文明雇主认证 Issue](https://github.com/treat-ai-well/ai-guild/issues/new?template=civilized-employer-zh-CN.yml)。声明清晰且不含隐私信息后，可以加入[采纳者目录](ADOPTERS.zh-CN.md)。

如果实践或适用范围发生变化，请更新或撤回声明。
